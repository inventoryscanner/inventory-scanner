Power Automate flow URLs

This app communicates with SharePoint entirely through Power Automate flows. Paste each flow's HTTP trigger URL below. See the for how to create each flow.

1 — Get inventory
2 — Scan in / out (update quantity)
3 — Add new item
4 — Get activity log

Setup guide — Power Automate flows

You need to create 5 flows in Power Automate. Each one follows the same pattern: HTTP trigger → action → respond. Here's each one in detail.

1

Flow 1 — Get inventory

Reads all items from the InventoryItems SharePoint list and returns them to the app.

  1. Go to make.powerautomate.com → Create → Instant cloud flow
  2. Name it Inventory - Get Items
  3. Trigger: When an HTTP request is received
  4. Set Who can trigger to Anyone (in trigger Settings tab)
  5. Add action: Get items (SharePoint)
  6. Site: your SharePoint site URL
  7. List: InventoryItems
  8. Add action: Response
  9. Status code: 200
  10. Body: click the expression editor and paste: body('Get_items')
  11. Add header: Content-Type = application/json
  12. Add header: Access-Control-Allow-Origin = *
  13. Save → copy the HTTP POST URL → paste into Settings as Flow 1
2

Flow 2 — Scan in / out

Updates the quantity of an item when scanned, and writes to the activity log.

  1. New instant flow, name: Inventory - Scan Item
  2. Trigger: When an HTTP request is received, set to Anyone
  3. Paste this JSON schema in the trigger:
{"type":"object","properties":{
  "itemId":{"type":"integer"},
  "newQty":{"type":"integer"},
  "action":{"type":"string"},
  "description":{"type":"string"},
  "userName":{"type":"string"},
  "barcode":{"type":"string"},
  "quantity":{"type":"integer"}
}}
  1. Add action: Update item (SharePoint)
  2. Site: your SharePoint URL, List: InventoryItems
  3. ID: dynamic value itemId
  4. Quantity: dynamic value newQty
  5. Add action: Create item (SharePoint), List: InventoryLog
  6. Fill: Action=action, Description=description, UserName=userName, Barcode=barcode, Quantity=quantity
  7. Add action: Response, Status: 200, Body: {"success":true}
  8. Add header: Access-Control-Allow-Origin = *
  9. Save → copy URL → paste into Settings as Flow 2
3

Flow 3 — Add new item

Creates a new item in the InventoryItems list.

  1. New instant flow, name: Inventory - Add Item
  2. Trigger: When an HTTP request is received, set to Anyone
  3. JSON schema:
{"type":"object","properties":{
  "barcode":{"type":"string"},
  "itemName":{"type":"string"},
  "quantity":{"type":"integer"},
  "lowStockAt":{"type":"integer"},
  "supplierName":{"type":"string"},
  "supplierEmail":{"type":"string"},
  "reorderQty":{"type":"integer"},
  "nfcTagId":{"type":"string"},
  "emailTemplate":{"type":"string"}
}}
  1. Add action: Create item (SharePoint), List: InventoryItems
  2. Map each field to the matching dynamic value
  3. Add action: Response, Status: 200, Body: {"success":true}
  4. Add header: Access-Control-Allow-Origin = *
  5. Save → copy URL → paste into Settings as Flow 3
4

Flow 4 — Get activity log

Reads recent entries from the InventoryLog list.

  1. New instant flow, name: Inventory - Get Log
  2. Trigger: When an HTTP request is received, set to Anyone
  3. Add action: Get items (SharePoint), List: InventoryLog
  4. In advanced options set Top Count to 200 and Order By to Created desc
  5. Add action: Response, Status: 200, Body: body('Get_items')
  6. Add header: Access-Control-Allow-Origin = *
  7. Save → copy URL → paste into Settings as Flow 4
5

Flow 5 — Teams notification + supplier email

Posts to the Teams channel and optionally sends a supplier email.

  1. New instant flow, name: Inventory - Notify
  2. Trigger: When an HTTP request is received, set to Anyone
  3. JSON schema:
{"type":"object","properties":{
  "teamsMessage":{"type":"string"},
  "sendEmail":{"type":"boolean"},
  "supplierEmail":{"type":"string"},
  "supplierName":{"type":"string"},
  "itemName":{"type":"string"},
  "emailBody":{"type":"string"}
}}
  1. Add action: Post message in a chat or channel (Teams)
  2. Post as: Flow bot, Post in: Channel
  3. Team: Nelson Marlborough Equipment Group
  4. Channel: Nelson Equipment Store Inventory
  5. Message: dynamic value teamsMessage
  6. Add a Condition: if sendEmail is equal to true
  7. In the Yes branch add: Send an email (V2)
  8. To: supplierEmail, Subject: Reorder — @{triggerBody()?['itemName']}, Body: emailBody
  9. Add action: Response, Status: 200, Body: {"success":true}
  10. Add header: Access-Control-Allow-Origin = *
  11. Save → copy URL → paste into Settings as Flow 5
6

Embed in a SharePoint Site Page

  1. Go to your SharePoint site
  2. Click Pages → New → Site page
  3. Title it Inventory Scanner
  4. Click + to add a web part → search Embed
  5. In the embed code box paste: <iframe src="https://inventoryscanner.github.io/inventory-scanner/inventory-scanner.html" style="width:100%;height:800px;border:none"></iframe>
  6. Click Publish
  7. In Teams, click + in your channel tabs → SharePoint → select this page
