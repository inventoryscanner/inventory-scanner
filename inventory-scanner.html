Setup guide — Power Automate flows

Create these 5 flows in Power Automate. Each uses an HTTP trigger so the app can call them directly.

1

Flow 1 — Get inventory

Reads all items from InventoryItems and returns them to the app.

  1. Go to make.powerautomate.com → Create → Instant cloud flow
  2. Name it Inventory - Get Items
  3. Trigger: When an HTTP request is received
  4. Click the trigger → Settings tab → set Who can trigger to Anyone
  5. Click + New step → search Get items → select SharePoint
  6. Site: your SharePoint URL  |  List: InventoryItems
  7. Click + New step → search Response
  8. Status code: 200
  9. Headers: add Content-Type = application/json and Access-Control-Allow-Origin = *
  10. Body: click the expression tab (fx) and type body('Get_items') → OK
  11. Save → copy the HTTP POST URL → paste in Settings as Flow 1
2

Flow 2 — Scan in / out

Updates item quantity and writes to the activity log.

  1. New instant flow → name: Inventory - Scan Item
  2. Trigger: When an HTTP request is received → set to Anyone
  3. Paste this in the Request Body JSON Schema box:
{"type":"object","properties":{
  "itemId":{"type":"integer"},
  "newQty":{"type":"integer"},
  "action":{"type":"string"},
  "description":{"type":"string"},
  "userName":{"type":"string"},
  "barcode":{"type":"string"},
  "quantity":{"type":"integer"}
}}
  1. Add step: Update item (SharePoint) → Site: your URL, List: InventoryItems, ID: dynamic itemId, Quantity: dynamic newQty
  2. Add step: Create item (SharePoint) → List: InventoryLog → map Action, Description, UserName, Barcode, Quantity to dynamic values
  3. Add step: Response → Status 200, Body: {"success":true}, Header: Access-Control-Allow-Origin = *
  4. Save → copy URL → paste in Settings as Flow 2
3

Flow 3 — Add new item

Creates a new item in InventoryItems.

  1. New instant flow → name: Inventory - Add Item
  2. Trigger: HTTP request → set to Anyone
  3. JSON schema:
{"type":"object","properties":{
  "barcode":{"type":"string"},
  "itemName":{"type":"string"},
  "quantity":{"type":"integer"},
  "lowStockAt":{"type":"integer"},
  "supplierName":{"type":"string"},
  "supplierEmail":{"type":"string"},
  "reorderQty":{"type":"integer"},
  "nfcTagId":{"type":"string"},
  "emailTemplate":{"type":"string"}
}}
  1. Add step: Create item (SharePoint) → List: InventoryItems → map all fields
  2. Add step: Response → Status 200, Body: {"success":true}, Header: Access-Control-Allow-Origin = *
  3. Save → copy URL → paste in Settings as Flow 3
4

Flow 4 — Get activity log

Reads recent entries from InventoryLog.

  1. New instant flow → name: Inventory - Get Log
  2. Trigger: HTTP request → set to Anyone
  3. Add step: Get items (SharePoint) → List: InventoryLog → in advanced options set Top Count: 200, Order By: Created desc
  4. Add step: Response → Status 200, Body: body('Get_items') (use expression tab), Header: Access-Control-Allow-Origin = *
  5. Save → copy URL → paste in Settings as Flow 4
5

Flow 5 — Teams notification + supplier email

Posts to Teams channel and optionally sends a supplier email.

  1. New instant flow → name: Inventory - Notify
  2. Trigger: HTTP request → set to Anyone
  3. JSON schema:
{"type":"object","properties":{
  "teamsMessage":{"type":"string"},
  "sendEmail":{"type":"boolean"},
  "supplierEmail":{"type":"string"},
  "supplierName":{"type":"string"},
  "itemName":{"type":"string"},
  "emailBody":{"type":"string"}
}}
  1. Add step: Post message in a chat or channel (Teams) → Flow bot, Channel, Team: Nelson Marlborough Equipment Group, Channel: Nelson Equipment Store Inventory, Message: dynamic teamsMessage
  2. Add step: Condition → sendEmail is equal to true
  3. In the Yes branch: Send an email (V2) → To: supplierEmail, Subject: Reorder request, Body: emailBody
  4. Add step (after condition): Response → Status 200, Body: {"success":true}, Header: Access-Control-Allow-Origin = *
  5. Save → copy URL → paste in Settings as Flow 5
